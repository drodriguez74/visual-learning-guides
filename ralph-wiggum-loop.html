The Ralph
Wiggum Loop

Named after the lovably clueless Simpsons kid who never gives up — a technique so simple it shouldn't work, yet it's shipping entire codebases overnight.

1
The Core Idea
"

Put a coding AI in a while loop. Feed it the same prompt. Let it fail repeatedly. Watch git collect the wins. It doesn't need to remember anything — your filesystem is its memory.

while true; do
  # Load plan + specs fresh every single iteration
  CONTEXT="$(cat fix_plan.md specs/stdlib/*)"

  # Run the AI agent on ONE task — chosen by the AI
  claude "$CONTEXT
  Choose the most important unfinished task.
  Implement it. Run tests. Commit if passing."

  # Brief pause to prevent runaway API costs
  sleep 2
done

That's literally it. No orchestration framework. No memory system. No complex agent graph. Just bash, a prompt, an AI CLI, and infinite stubbornness. Deterministically dumb in an undeterministic world.

The Context Window Secret

The magic is what you put in the window each loop. Pick ONE task. Keep the window lean. The more you cram in, the less the AI can "see" clearly — like trying to read a book through a keyhole.

4
What Happens Each Iteration
1
📋
Load the Stack
Fresh load of fix_plan.md and all spec files. These are deterministically allocated every single loop — always in the same slots. No surprises.
2
🤔
AI Picks ONE Task
The AI reads the plan and specs, then picks the single most important unimplemented thing. You trust it to decide. Yes, really. Full hands-off.
3
⌨️
Implement It
The AI writes the code, creates files, edits existing ones. It works inside the actual repo — real files, not hypotheticals.
4
🧪
Run the Tests
Tests run immediately. If they pass — great, commit. If they fail — the AI sees the error output and tries to fix within this same turn. Fail fast, retry fast.
5
✅
Commit or Abandon
Passing? Git commit. Progress is real and permanent. Failing still? Abandon the attempt, move on. The next loop will try something different.
6
🔁
Loop Again
Back to step 1. Fresh context. New task choice. The git history grows. The codebase builds itself — overnight, unsupervised, persistently.
5
Why "Dumb" Wins
❌ Sophisticated Agent
💔Complex memory graph gets corrupted over time
💔State machines fail in unexpected edge cases
💔Errors compound — one bad step poisons everything downstream
💔Debugging the orchestration is harder than the code
💔Context compaction silently loses critical decisions
💔Lots of engineering to build the agent itself
✅ Ralph Loop
💚No memory to corrupt — fresh every iteration
💚No state to corrupt — bash has no state
💚Bad iterations are abandoned, not propagated
💚Zero framework — debug the AI's output, that's it
💚Each loop sees the current real filesystem state
💚~10 lines of bash — anyone can understand it

Ralph Grows Up
Gas Town
Multi-agent orchestration
If Ralph is one agent looping, Gas Town is a whole city of Ralphs. Multiple parallel Ralph loops, each picking non-overlapping tasks. Described as "Kubernetes for agents." Invented by Steve Yegge. An entire autonomous development assembly line.
Ralph TUI
Visibility layer
A terminal UI that watches the Ralph loop in real-time — shows what the AI is doing, what it committed, what it abandoned. Builds human trust in the autonomous process by making the "black box" visible.
MEOW Tasks
Molecular Expression of Work
Breaking down a PRD into tasks so granular they can be atomically picked, executed, and committed by a single Ralph iteration. The smaller the task definition, the higher Ralph's success rate per iteration.
Co-Reg Loop (CRDAL)
Academic extension (2026)
Researchers added a "Metacognitive Co-Regulation Agent" — a second AI that watches Ralph's choices and prevents design fixation. Proven to generate better solutions without meaningfully increasing compute cost.
