Orchestration
Chain-of-Thought Pipeline
Decompose complex tasks into sequential reasoning steps, each passing context forward
+
Live Flow Diagram
What it does
Routes a complex prompt through a fixed sequence of LLM calls — each step narrows the problem, enriches context, or transforms the output before passing it to the next stage. The chain is deterministic and auditable.
When to use
  • Tasks with distinct phases: parse → reason → format
  • When intermediate outputs need human review
  • Compliance workflows needing audit trail per step
  • Latency-tolerant, accuracy-critical pipelines
Strengths
Transparent reasoning · Easy to debug · Auditable steps · Predictable cost
Trade-offs
Error propagation · Sequential latency · Rigid structure
// Step 1: extract intent
const intent = await llm(`Extract the user's intent: ${input}`)
// Step 2: enrich with context
const enriched = await llm(`Given intent: ${intent}, retrieve relevant facts`)
// Step 3: generate response
const output = await llm(`Respond using: ${enriched}`)
Orchestration
Orchestrator / Sub-Agent
A planning agent spawns specialized sub-agents, coordinates results, synthesizes output
+
Live Flow Diagram
What it does
A high-level orchestrator LLM receives a goal and decomposes it into sub-tasks. Each sub-task is delegated to a specialized sub-agent (coder, researcher, reviewer). Results flow back up; the orchestrator synthesizes the final answer.
When to use
  • Complex multi-domain problems (code + research + QA)
  • Tasks that benefit from parallel workstreams
  • When sub-agents need different tools or permissions
  • Claude Code's spawn pattern for large codebases
Strengths
Specialization · Parallelism · Separation of concerns · Scales complexity
Trade-offs
Orchestrator is a bottleneck · Context handoff cost · Hard to debug
// Orchestrator decomposes goal
const plan = await orchestrator.plan(goal)
// Spawn parallel sub-agents
const results = await Promise.all(
  plan.tasks.map(t => spawnAgent(t.role, t.task))
)
// Synthesize
return await orchestrator.synthesize(results)
Retrieval
Retrieval-Augmented Generation (RAG)
Inject relevant retrieved knowledge into LLM context at inference time
+
Live Flow Diagram
What it does
At query time, the user's input is embedded and used to retrieve semantically similar documents from a vector store. Retrieved chunks are injected into the LLM prompt as grounding context before generating a response.
When to use
  • Private or proprietary knowledge bases
  • Reducing hallucination in factual domains
  • Content that changes faster than retraining
  • DisputesGPT policy and procedure retrieval
Strengths
Grounded answers · Updatable knowledge · Source citations possible
Trade-offs
Retrieval quality ceiling · Chunking strategy matters · Context window pressure
const embedding = await embed(userQuery)
const chunks = await vectorStore.search(embedding, k=5)
const prompt = `Context:\n${chunks.join('\n')}\n\nQ: ${userQuery}`
return await llm(prompt)
Routing
Semantic Router
Classify incoming requests and dispatch to the best-fit model, agent, or handler
+
Live Flow Diagram
What it does
A lightweight classifier (LLM or embedding similarity) inspects each incoming request and routes it to a specialized downstream handler — fast model for simple queries, expensive model for complex ones, human for edge cases.
When to use
  • Cost optimization: cheap model for 80% of traffic
  • Multi-domain systems (fraud vs. disputes vs. payments)
  • Hard/soft fallback chains
  • Compliance routing: sensitive → human-in-loop
Strengths
Cost efficiency · Specialization · Simple to extend · Observable
Trade-offs
Router errors cascade · Boundary cases misclassified · Extra latency
const intent = await classifier.classify(query)
const handler = routes[intent] ?? routes['default']
return await handler.handle(query, context)
Evaluation
LLM-as-Evaluator
A second LLM judges the first's output and triggers correction or retry
+
Live Flow Diagram
What it does
The generator LLM produces a draft. An evaluator LLM (same or different model) scores it against criteria: correctness, tone, completeness, policy compliance. Below-threshold outputs are sent back for revision with specific feedback attached.
When to use
  • Customer-facing content requiring quality floor
  • Regulated outputs (dispute letters, compliance text)
  • When human review doesn't scale but quality matters
  • Automated regression testing of AI pipelines
Strengths
Quality floor enforcement · Self-correcting · Scalable review
Trade-offs
2× token cost minimum · Evaluator can be wrong too · Retry loops risk infinite
let draft = await generator.generate(prompt)
let attempts = 0
while (attempts++ < MAX_RETRIES) {
  const score = await evaluator.score(draft, rubric)
  if (score >= threshold) break
  draft = await generator.revise(draft, score.feedback)
}
Memory
Layered Memory Architecture
Combine in-context, episodic, and semantic memory for persistent, context-aware agents
+
Live Flow Diagram
What it does
Agents maintain three memory tiers: in-context window (immediate), episodic store (past conversations, vector-indexed), and semantic store (distilled facts). Each query selectively retrieves from all three layers before responding.
When to use
  • Long-running agents that span multiple sessions
  • Customer service bots needing full account history
  • Any agent where "remember our last conversation" matters
  • DisputesGPT case history and analyst preferences
Strengths
Persistent personalization · Long-horizon coherence · Efficient context use
Trade-offs
Stale memory risk · Privacy surface area · Retrieval latency adds up
const context = {
  immediate: conversationBuffer,
  episodic: await episodicStore.recall(userId, query),
  semantic: await semanticStore.lookup(entities)
}
return await agent.respond(query, context)
Safety
Input / Output Guardrails
Intercept, validate, and sanitize at both entry and exit — independent of the LLM
+
Live Flow Diagram
What it does
A deterministic validation layer wraps the LLM on both sides. Input guardrails strip PII, detect injection attacks, enforce topic scope. Output guardrails check for policy violations, hallucinated references, and format correctness before the response reaches the user.
When to use
  • Any customer-facing production AI system
  • PCI / GDPR environments — PII must not enter LLM context
  • Financial advice where hallucinated numbers are dangerous
  • Multi-tenant systems where prompt injection is a threat vector
Strengths
Deterministic control · Compliance-auditable · LLM-agnostic layer
Trade-offs
False positives block valid requests · Rules lag threats · Double latency
// Input layer
const safe = inputGuard.sanitize(userInput) // strips PII, detects injection
const raw = await llm(safe.cleanedPrompt)
// Output layer
const result = outputGuard.validate(raw) // policy check, format verify
return result.approved ? result.text : fallback
Orchestration
Parallel Exploration + Voting
Run N agents simultaneously on the same problem, synthesize or vote on the best answer
+
Live Flow Diagram
What it does
The same task is dispatched to N parallel agents simultaneously. Each agent works independently. A voting or synthesis step picks the majority answer, best-scored output, or merges non-contradictory results. Reduces variance, increases reliability.
When to use
  • High-stakes decisions where variance is unacceptable
  • Code generation where multiple solutions can be tested
  • Creative tasks where diversity improves output quality
  • Claude Code's parallel explore agents for large refactors
Strengths
Lower variance · Higher confidence · Parallelism hides latency
Trade-offs
N× token cost · Voter/synthesizer is a new failure point · Overkill for simple tasks
const N = 3
const drafts = await Promise.all(
  Array(N).fill(prompt).map(p => llm(p, { temperature: 0.8 }))
)
// Majority vote or evaluator picks winner
return await synthesizer.pick(drafts)
