01 — The Shift

From Reactive to Agentic

Traditional AI waits for a prompt and returns a response. Agentic AI takes ownership of a goal and works toward it through multiple steps.

⬛ Reactive AI

  • 1 User provides a complete, specific prompt
  • 2 Model generates a single response
  • 3 Conversation ends or user re-prompts
  • 4 No persistent memory between turns
  • 5 No access to external tools or systems

⚡ Agentic AI

  • → User provides a high-level goal
  • → Agent breaks it into sub-tasks autonomously
  • → Executes, observes results, adjusts plan
  • → Maintains memory across steps
  • → Calls external tools: search, code, APIs

03 — Superpowers

Tools Give Agents Reach

Agents extend beyond language by calling external tools — turning reasoning into real-world action.

💻

04 — How It Works

A Goal Becomes Action

Trace a single agentic run from user intent to completed task.

User states a goal
"Research competitors in our space and draft a SWOT summary."
TRIGGER
Agent decomposes the task
Breaks the goal into: (1) identify competitors, (2) gather data per competitor, (3) synthesize into SWOT framework.
PLAN
Tool calls begin
Searches the web, fetches company pages, reads financial filings, extracts relevant signals.
ACT
Observes & adjusts
One competitor has sparse data. Agent pivots: searches for press releases and LinkedIn instead.
OBSERVE
Memory accumulates
All gathered facts are held in working context so no step is repeated and the final synthesis is coherent.
MEMORY
Goal achieved — output delivered
A structured SWOT document is produced, grounded in live data, ready for the user to review.
DONE

05 — Real Examples

Agents in the Wild

High-level goals, multi-step execution.

🧑‍💻
Software Engineering
"Fix the failing tests in the auth module."
  • Read failing test output
  • Locate relevant source files
  • Identify root cause in code
  • Write & apply patch
  • Re-run tests to verify fix
📊
Data Analysis
"Analyze last quarter's sales and flag anomalies."
  • Pull data from warehouse API
  • Write & run analysis script
  • Identify statistical outliers
  • Cross-reference with calendar events
  • Generate annotated report
🔬
Research
"Summarize recent papers on RAG improvements."
  • Search arXiv & semantic scholar
  • Filter by recency & citations
  • Extract key contributions
  • Cluster by technique
  • Write structured literature summary
🏦
Finance Ops
"Triage incoming support cases by priority."
  • Fetch open tickets from ServiceNow
  • Classify by type & urgency
  • Check SLA deadlines
  • Route to correct queue
  • Draft initial response drafts
