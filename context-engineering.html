Six Pillars of Context Engineering
01
01 — SYSTEM PROMPT

Identity & Rules

Define persona, tone, constraints, and output format. The model's "job description" — read before every token generated.

02
02 — MEMORY

State Injection

Long-term facts retrieved from vector stores or databases, injected as structured context so the model "remembers" across sessions.

03
03 — TOOLS

Capability Schemas

Function definitions tell the model what actions it can take — search, write code, call APIs. Well-named tools reduce ambiguity.

04
04 — HISTORY

Conversation State

Prior turns give coherence. Engineering this means truncating smartly, summarizing older turns, and preserving key decisions.

05
05 — RAG

Knowledge Retrieval

Embed queries, fetch semantically relevant chunks, and inject them as grounding evidence — reducing hallucinations dramatically.

06
06 — USER QUERY

Input Shaping

Pre-process, rewrite, or augment the user's raw message before it hits the model — adding examples, constraints, or format hints.

Engineering the Context Pipeline
1
Input

Receive & Parse User Message

Capture raw input, detect intent, extract entities. Decide what type of context retrieval is needed.

2
Retrieval

Fetch Relevant Context

Query vector DB for semantic matches, pull user memory records, load applicable tool schemas based on detected intent.

3
Assembly

Construct the Context Window

Pack layers into a finite token budget: system → memory → tools → summarized history → retrieved docs → user message. Order matters.

4
Inference

Model Generates Response

The LLM reasons over the assembled context, potentially invoking tools (which add more context in a loop) before final output.

5
Output

Post-process & Update State

Parse structured response, update memory stores, log the turn, and feed back into context for the next interaction.

Context Quality: Bad vs. Good
✗ Poor Context Engineering
Vague system prompt with no constraints
Full untruncated conversation history
All tools injected regardless of intent
No retrieved knowledge, just raw query
User message passed verbatim, unprocessed
No token budget management
✓ Strong Context Engineering
Precise system prompt with examples & format
Smart history summarization past N turns
Only relevant tool schemas injected
Top-K semantically ranked RAG chunks
Query rewriting and intent enrichment
Token budget allocated per layer
