EST. 1964 — ENGINEERING APPS
PL/I
Programming Language One
IBM's own language — more powerful than COBOL for systems programming, more structured than assembler. Used in telecom billing, scientific computing, and any app requiring complex data structures, recursion, and pointer manipulation alongside business logic.
Telecom Billing Scientific Apps Complex Data Compiler Tools
/* PL/I — Bill computation */ BILLING: PROC OPTIONS(MAIN); DCL ACCOUNT_ID CHAR(10); DCL BALANCE DEC FIXED(15,2); DCL RATE DEC FLOAT(6); GET LIST(ACCOUNT_ID, BALANCE); RATE = 0.0325; PUT SKIP LIST( BALANCE * (1 + RATE)); END BILLING;
EST. 1952 — METAL-LEVEL POWER
HLASM
High Level Assembler (z/Architecture)
Direct access to z/Architecture instructions. Used for performance-critical kernel modules, cryptographic routines, I/O drivers, and anything that must squeeze every nanosecond. Still written today for interrupt handlers and licensed program internals.
OS Internals Crypto Routines I/O Drivers Performance Libs

EST. 1979 — SCRIPTING KING
REXX
Restructured Extended Executor
The shell scripting language of z/OS. Glues everything together — runs JCL jobs, parses SYSOUT, calls ISPF panels, invokes DB2, and automates operations tasks. Every mainframe operator writes REXX. Simple, readable, incredibly powerful in the z/OS environment.
Automation JCL Management ISPF Scripting Ops Tooling

EST. 1983 — DATA POWERHOUSE
DB2 SQL
DB2 for z/OS — Embedded & Dynamic SQL
Not a language per se, but SQL embedded in COBOL, PL/I, Java, and C programs via the DB2 precompiler. Static SQL is bound at compile time into access plans — giving sub-millisecond execution on trillion-row tables. Features not found in any other RDBMS.
Embedded SQL Stored Procs Temporal Tables Row-level Security

§ 04 — MIDDLEWARE ECOSYSTEM
KEY PLATFORM COMPONENTS
Component Category What It Does Used With
CICS TP MONITOR Customer Information Control System. The world's highest-throughput transaction manager. Handles session management, task dispatching, and resource recovery for millions of concurrent OLTP transactions. Bridges COBOL programs to networks. COBOL, PL/I, Java, C
DB2 z/OS RDBMS IBM's relational database for mainframe. Tightly coupled to hardware — uses dedicated data engines and z/Architecture instructions. Row-level access control, temporal tables, and encryption that cloud RDBMS vendors still can't match in throughput/cost. COBOL, Java, PL/I, SQL
IMS HIERARCHICAL DB + TP Information Management System — a hierarchical database AND transaction manager, predating SQL. Many 1960s data structures still live here. Airlines, banks, and telcos have IMS data they can't easily migrate. Incredibly fast for known-path queries. COBOL, Assembler, Java
MQ Series MESSAGING IBM MQ provides persistent, transactional message queuing between mainframe and distributed systems. Guarantees exactly-once delivery. The glue between COBOL batch jobs, Java microservices, and cloud-native consumers. Java, COBOL, C++, .NET
z/OS Connect API GATEWAY Exposes COBOL programs, CICS transactions, and DB2 stored procs as RESTful JSON APIs — without rewriting any COBOL. A mobile app calls a REST endpoint; z/OS Connect forwards it to a 40-year-old COBOL program. The modernization bridge. REST, JSON, COBOL, CICS
RACF SECURITY Resource Access Control Facility. Controls who can touch every dataset, program, port, and transaction — down to the record level. Audit trails for every access. NIST compliance baked in. Far more granular than AWS IAM. All languages, z/OS kernel
JES2 / JCL JOB SCHEDULER Job Entry Subsystem manages batch job submission, scheduling, output routing, and spool management. JCL (Job Control Language) defines job steps, datasets, and dependencies. The backbone of nightly batch — end-of-day clearing, statement generation, report runs. COBOL, PL/I, REXX, all
WebSphere Liberty JAVA RUNTIME Lightweight Java EE / Jakarta EE runtime on z/OS. Runs Spring Boot, JAX-RS, microservices — offloaded to zIIP specialty processors to minimize general CP consumption and licensing costs. Runs in containers via z/OS Container Extensions (zCX). Java, Jakarta EE, Spring
